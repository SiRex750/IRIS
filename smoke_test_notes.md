FEASIBILITY SIGNAL: valid PyTorch CPU numbers trend 74s (SmolVLM2 2.2B, fp32) -> 467s (Qwen2.5-VL 3B, bf16) — capable PyTorch captioning is impractical on CPU on this hardware. DECIDING RESULT: quantized llama.cpp via Ollama (moondream:1.8b, Q4_0) breaks this wall entirely at ~13.12s/frame — ~5.6x faster than SmolVLM2 and ~35x faster than Qwen, with comparable non-confabulation quality (0/40 diff-language). Quantized llama.cpp runtimes, not PyTorch/transformers, are the practical path for CPU-only captioning on this hardware.
